Docker para developers que no son DevOps

Una guía práctica para entender y usar Docker en tu workflow sin tener que aprender Kubernetes. Dejá de pelear con "en mi máquina funciona".

Imagen de portada del artículo

Uno de los mayores dolores de cabeza cuando empezamos a trabajar en equipo o desplegamos software es la famosa frase: "En mi máquina sí funciona". Diferentes versiones de Node, bases de datos configuradas a medias, o dependencias del sistema que faltan.

¿Qué es realmente un contenedor?

Para no meterte tecnicismos aburridos: un contenedor es como una caja que empaqueta tu código y todo lo que ese código necesita para ejecutarse de manera aislada. Si corre en tu PC, va a correr exactamente igual en el servidor.

"Docker no reemplaza a las máquinas virtuales, las hace obsoletas para el desarrollo de aplicaciones del día a día."

El archivo clave: Dockerfile

Para definir cómo se construye tu entorno, usás un archivo plano llamado Dockerfile. Acá tenés un ejemplo básico para una app de Python:

FROM python:3.10-slim
      WORKDIR /app
      COPY requirements.txt .
      RUN pip install -r requirements.txt
      COPY . .
      CMD ["python", "main.py"]

Docker Compose: Tu mejor aliado

Cuando tu app necesita una base de datos (por ejemplo, PostgreSQL), levantar todo a mano con comandos largos es tedioso. Ahí entra Docker Compose, permitiéndote levantar toda tu infraestructura local con un simple comando:

docker compose up -d

Conclusión

Como desarrollador, no necesitás ser un experto en orquestación ni dominar Kubernetes para sacarle provecho a Docker. Con entender cómo aislar tu entorno local y automatizar la base de datos de desarrollo ya tenés el 80% del beneficio asegurado.
